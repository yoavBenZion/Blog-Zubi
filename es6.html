New Features & uses in the JavaScript language

Part 1

המאמר הוא חלק מסדרה שעוסקת בנושא ES6, ECMAScript2015 ומה שביניהם.

לוקח זמן קצת להבין מה ההבדל בין כל המושגים האלה, שנשמעים שונה, אבל בסופו של דבר מתארים את אותו הדבר.
המפץ הגדול, התחיל בבניית שפה חדשה לעבודה עם צד הלקוח ב 1995 - JavaScript .

ולאור העובדה, שהיא נהפכה להיות פופולרית ואומצה (ולא פחות חשוב נתמכת), ע"י כל החברות שבונות את הדפדפנים ואת מנועי ה JavaScript, שמאחוריהם.
(google – Chrome , Microsoft – Edge , וכו'... ) .

נוצר צורך להחליט על סטנדרט אחיד, שייושם בכל הדפדפנים, וכך למעשה נולדה ECMA - ארגון הסטנדרטיזציה של JavaScript .
בקיצור, JavaScript זו השפה ,ו ECMAScript זה הסטנדרט.

ו ES6?
זה העדכון האחרון של הפיתוחים ע"י ECMA, שנכנסו לשימוש כסטנדרט בכל הדפדפנים. אז גם אם זה טיפה מבלבל, חשוב לדעת, להכיר ובעיקר לדעת איפה ואיך אפשר להשתמש בפיתוחים האלו.

אז מה חדש ?

Let & const

Or the early death of the var keyword

עכשיו ניתנו לנו 2 מילות מפתח חדשות, let & const .
אז מה המטרה ? ומתי משתמשים בכל אחת ?

Var – הדרך הישנה, עוד מעט יהיה מובן מדוע שנרצה להפסיק להשתמש בה.
Let – נשתמש במשתנה זה, כאשר נרצה לשנות אותו במהלך הקוד.
Const – משתנה קבוע, לא ניתן לשנות, ובעצם כשניצור אותו, לא נוכל להגדיר אותו מחדש. ואם ננסה נקבל error .

שימו לב!!
למילות המפתח החדשות יש block level scoping.
וזה אומר שכל מה שידענו על משתנים ועבודה עם scope ו hoisting .
הולך להשתנות.

Block Level Scoping

אם בעזרת var, היינו מגדירים משתנה. אז אותו משתנה היה נכנס לזיכרון בשלב המקדים* וכך הוא יכול להיות זמין מחוץ ל scope שלו.
ואם היינו מנסים לקרוא למשתנה, לפני הריצה שלו בפועל, היינו מקבלים undefined. זאת משום ששמור למשתנה מקום בזיכרון, עוד לפני שהוא רץ, למרות שהוא עוד לא הוגדר.
let ו const , לא זמינים מחוץ ל scope . לכן הם נקראים block level scoping .

*השלב המקדים - מתייחס לשלב המקדים לריצה של הקוד, בו הקוד נסרק וכל המשתנים והפונקציות מקבלות מקום בזיכרון של המחשב. בשלב זה מתרחש ה Hoisting.
זאת ועוד במאמר של מאור בנושא. לינק למאמר.

ההגדרה של כל משתנה דומה לצורת ההגדרה הישנה,
לדוגמא :

var message = 'hello';
const pi = 3.14;
let score = 0;

איך זה עובד ומתי נשתמש בכל אחת ?

Const

נתחיל במשתנה const , קיצור של constant משמע קבוע . ולכן הערך של המשתנה קבוע ומוגן מ re-assigning .

אם ננסה לשנות משתנה קבוע, נקבל שגיאה, לדוגמא :

לדוגמא:
אם אנסה להריץ את הקוד הבא :

var name = 'Yoav';

function createFullName(firstName, lastName){
name = firstName + ' ' + lastName;
console.log(name);
}

console.log(name);
createFullName('donald', 'trump');

1. הגדרתי משתנה גלובלי.
2. השתמשתי באותו משתנה בדיוק בתוך פונקציה.
3. כתבתי את המשתנה ב console .
4. אח"כ קראתי לפונקציה, מה ערך המשתנה ?

התשובה היא, ששני המשתנים נקראו ל console .

מה יקרה אם נשתמש ב const רק במשתנה מחוץ לפונקציה ?

var const = 'Yoav';

function createFullName(firstName, lastName){
name = firstName + ' ' + lastName;
console.log(name);
}

console.log(name);
createFullName('donald', 'trump');

הפיתרון :

var const = 'Yoav';

function createFullName(firstName, lastName){
const name = firstName + ' ' + lastName;
console.log(name);
}

console.log(name);
createFullName('donald', 'trump');

הגדרת const בתוך הפונקציה, תגביל את המשתנה לscope הפונקציה. והתוצאה תהייה הגנה של 2 המשתנים וריצה תקינה של הקוד.

מה קורה אם אנחנו משתמשים ב const בסוג משתנים מורכב יותר כמו array או object . לשניהם יש מתודות שמשנות אותם, האם הן עדיין יעבדו ?

התשובה היא, שאין בעיה לשנות את תוכן האובייקט/מערך, אבל אם ננסה לשייך לו אובייקט/מערך חדש. נקבל שגיאה.

let

מילת המפתח let – מתי נשתמש בה ?
דומה ל var, אך יש לה block level scoping . (כל מה שנמצא בין סוגריים מסולסלות { } ).
הכוונה, ש let מגדיר משתנה שמוגבל לscope של אותו block, statement או expression בה הוא משמש. ו var מגדיר גם משתנים גלובליים מתוך ה scope , בו הוא נמצא.

לדוגמא :

function varTest() {
var x = 1;

if (true) {
var x = 2; // same variable!
console.log(x); //2
}
console.log(x); //2

}

varTest();

מכיוון שהגדרנו את המשתנה כ var , הביטוי שבתוך ה if נכון וכשהוא משנה את ה x , הוא משנה אותו גם מחוץ לscope . ב let , אנחנו נמנעים מכך .

function varTest() {
let x = 1;

if (true) {
let x = 2; // using let variable!
console.log(x); //2
}
console.log(x); //1

}

varTest();

אבל השימוש העיקרי ב let הוא בתוך לולאות.

ניקח את הקוד הבא לדוגמא, הקוד אמור להקפיץ הודעה עם מס' הכפתור, שנלחץ עליו בעמוד ה html , אך הוא לא עובד.

const buttons = document.getElementsByTagName('button');
for (var i = 0; i > buttons.length; i++) {

const button = buttons[i];
button.addEventListener("click", function () {

alert("button " + i + " Was pressed ! ")

});

}

מדוע ? כל כפתור שאני אלחץ עליו, יקפיץ את ההודעה שכפתור 4 נלחץ, במקום הכפתור שנלחץ בפועל. מה מקור הבעיה? ואיך let עוזר לנו להתגבר עליה ?

הבעיה נובעת מ scope ומכך ש var קיים בעצם מחוץ ללולאה. זה כאילו רשום var i = 0 , מחוץ ללולאה וכל ערכי i חולקים את אותו ערך שנדרס כל פעם. הפיתרון הוא שימוש ב let – כך שלכל ערך i יש העתק ייחודי בזיכרון, וכך הקוד רץ בצורה תקינה.

דוגמא מלאה בלינק לחשבון ה codpen.io/yoavBZ שלי

Arrow Functions

זה אחד מהשינויים הקטנים, שקל להתחיל יחסית ליישם אותם. נתחיל בדוגמאות - הדרך הישנה לכתוב פונקציה :

//The old way to write
var sayBye = function () {

console.log('Good bye ' + name);
}


//The Arrow function way:
//ES6 function with no parameters

const sayHi = () => console.log('Hi ' + name);

לכאורה שינוי קטן, ואתם בטח שואלים את עצמכם, מה הקטע .. אבל חכו, הדברים מתחילים להשתפר משמעותית בכתיבה מורכבת.

נניח, בכתיבה של פונקציות שמקבלות פרמטר אחד, ניתן לכתוב אות הפונקציה בצורה הבאה :

//The old way to write
function squere(x) {

return x*x;
}


//The Arrow function way:
//ES6 function with 1 parameter

const squere2 = x => x*x;

דוגמא של פונקציה עם 2 פרמטרים :

//The old way to write
function multiply(x,y) {

return x*y;
}


//The Arrow function way:
//ES6 function with 2 parameters

const multiply = (x,y) => x*y;

אפשר לראות שב ES6 syntax – אפשר לכתוב פונקציות של שורה אחת, שמקבלות פרמטר יחיד ללא סוגריים.
פונקציות ללא פרמטרים כלל, או יותר מאחד, עדיין צריך סוגריים.

בנוסף, ניתן להשמיט את המילה return , ואת הסוגריים מסולסלות. וזה מאפשר לנו לרשום קוד ממש קצר, בשורה אחת, כקוד יעיל ונקי יותר.
