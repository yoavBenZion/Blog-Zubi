המנגנון שיתמרן לכם את הקוד

במאמר זה אתייחס לשני מנגנוני יעול שנכתבים תחת לולאת IF ויכולים להיות ההבדל הדק שבין 5 שורות קוד מגושמות שיכולות לגרום לקוד לקרוס לבין שורת קוד אחת שתציל את הקוד שלכם מקריסה ותייעל את קצב העבודה שלכם.

לפני שנתחיל, מהו BOOLEAN? ערך שמסתכם בשתי צורות: True/False. ישנם מספר ערכים בוליאנים אשר נותנים לנו תשובת False:

  • False / Null / Undefined / NaN / " " / ' ' / 0

ומספר ערכים בוליאנים לTRUE:

  • True
  • כל מספר מלבד 0
  • כל מחרוזת (STRING) שהיא לא ריקה
  • NaN
  • כל סוג של מערך (ARRAY []) אפילו אם הוא ריק
  • כל סוג של אובייקט (OBJECT {}) אפילו אם הוא ריק

לפני שנבין מה עושים עם כל אלו חשוב שנבין שכל מחרוזת שנרצה להמיר לערך בוליאני נעשה זאת על-ידי שימוש בנוסחאת converting to boolean:
'string'! / 'string'!! / Boolean('סטרינג') / (false) / (true) /

כמה הערות חשובות:
** מחרוזת שלפניה יהיה ! תהיה FALSE
** מחרוזת שלפניה יהיו !! תישלח כFALSE ותחזור כTRUE
** מחרוזת בתוך (מחרוזת) עם המילה BOOLEAN לפניה תחזור כTRUE

אז למה OPERATOR? המנגנון נותן לכם את האפשרות לרשום את הקוד בצורה הרבה יותר פשוטה, הרבה יותר יעילה למחשב לקריאת הקוד וכמובן תציל את הקוד שלכם מלקרוס כשלא הגיע הערך.
המנגנונים הללו משתמשים בערכים בוליאנים (Booleans) אך להבדיל מפונקציה פשוטה שתגרום לקוד שלכם לקרוס במידה ולא קיבלתם ערך, המנגנונים הללו יתנו לנו את האפשרות לקבל "או או (||)" או "גם וגם (&&)" וכך לשמור על הקוד פעיל. אסביר... לדוגמא, נכתוב את הפונקציה הבאה:
בפונקציה הזו ביקשנו להחזיר את שם המשתמש אם המשתמש ביצע Login למערכת. את אותה פונקציה נוכל לכתוב עם GUARD OPERATOR (&&):
מה זה נותן לנו? הGUARD הוא מנגנון שמבצע שתי פעולות שונות על-ידי שתי משתנים שונים בתלות בין השניים.
במידה ובקשה א' (לדוגמא; חיבור למערכת) מתקבלת אז יעבור המנגנון לבקשה ב' (לדוגמא; החזרת שם המשתמש) ולכן נקבל True.
במידה והמשתמש לא היה מתחבר למערכת(ז"א שהערך שLOGIN היה מקבל FALSE) המנגנון לא היה עובר לבקשה ב' ולכן היינו מקבלים תוצאת False.
התנהגות המנגנון היא תלותית. הוא מתחיל בקבלת תשובה מהמשתנה השמאלי שהצבנו ובמידה וקיבלנו ערך False כל הפונקציה תחזיר False.
במידה וקיבל תשובה True מהערך השמאלי, רק אז יעבור לערך הימני ובמידה ויקבל תשובה חיובית גם מן המשתנה הימני הפונקציה תיתן לנו True.
בין שני המשתנים תמיד יוצב הסימן שיוצר את מנגנון הפונקציה ואת התלות שביניהם.

מנגנון נוסף שנדון בו הוא ה: Default OperatOR (||):
לעומת מנגנון ה &&, המנגנון ה'דיפולטיבי' (מסומן על-ידי ||) הוא מנגנון א-תלותי
שבו אנו נציב שני ערכים או יותר ואנו נקבל את הערך הראשון אשר מתקבל כTRUE כאשר המנגנון מתחיל את קריאת הבקשה מצד שמאל. נדגים:

הגדרנו ערך שם וביקשנו שייתן לנו ערך א' (NAME),
הצבנו את סימון המנגנון || ורשמנו שבמידה ולא מתקבל true, יתקבל ערך ב' (שיציג על המסך No name assigned).
יכולנו להציב כמה ערכים שנרצה במנגנון. התועלת של אפשרות זו היא לתת לקוד שלכם כמה שיותר אפשרויות על-מנת שיקבל מצב TRUE ולא יקרה מצב שבו נקבל false או undefined..
שילוב בין השניים:
דבר אחרון שחשוב מאוד להבין לגבי שני המנגנונים הללו הוא שניתן לשלב ביניהם. לדוגמא:

בדוגמא העליונה הוגדרו 3 משתנים a,b,c
לאחר מכן פתחנו לולאת IF שבה בתחילה הצבנו את a ו-b בתלות תחת מנגנון guard.
לאחר מכן הצבנו את מנגנון OR כנגד משתנה c (משול לסדר פעולות חשבוניות שבו "()" יהיו קודמות לכפל וחילוק).
זאת-אומרת, שאם במנגנון הGUARD שתי המשתנים היו TRUE, מנגנון הOR היה מקבל את התשובה הראשונה אך מכיוון שאחד המשתנים False, מנגנון הOR עבר לתשובה השנייה וקיבלנו את התוצאה הרצויה ב Console.
