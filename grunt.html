Grunt - JavaScript Task Runner

המאמר הבא יעסוק בחבילה שימושית מאוד שנמצאת בתוך NODE.JS בשם grunt. מדובר בחבילה שמדורגת בין עשרת המקומות הראשונים מבחינת הורדות מדי חודש.
אבל,איך זה יכול לעזור לנו בתהליך הפיתוח ?

Grunt מאגדת בתוכה מלא תוספים שימושיים שנכתבו עבורנו המקלים עלינו כמתכנתים בביצוע פעולות שיגרתיות ואוטומטיות כמו:

JSHINT- תוסף שיודע לאתר SYNTAX ERROR ובעיות שעלולות והיווצר בקוד שלכם.
MINIFY CSS/HTML- תוסף שמקטין את נפח הקבצים לשיפור ביצועים (זמן טעינת אתרים).
קימפול קבצי sass לcss - כשחל שינוי בקובץ sass אוטומטית מקמפל לcss.
Watch - מעקב אחרי שינוי כלשהו בקבצים.

את המשימות שניתן לשרת שלנו אנחנו נגדיר בתוך קובץ Gruntfile.js שעליו ארחיב בהמשך המאמר, הוא יכיל את כל ההגדרות של התוספים בהם אנחנו נשתמש בשרת שלנו.

על מנת שנוכל להתקדם, חשוב שיהיה לכם קצת מושג מהו nodejs וכמובן שכבר התקנתם Node.js והרמתם שרת מקומי שעליו תוכלו לעבוד.

במאמר זה אדבר בעיקר על 2 התוספים הבאים:

1.grunt-contrib-watch - התוסף עוקב אחרי שינויים בקבצים שנמצאים בשרת שלנו, אנחנו כמובן יכולים לשלוט באיזה תיקיה ואיזה סוגי קבצים הוא יעקוב, במידה וחל שינוי כלשהו, נוכל להגדיר לו לבצע פעולה מסויימת על אותו קובץ.
2.grunt-contrib-htmlmin - תוסף שמכווץ את קבצי הhtml, לצורך שיפור בביצועים בזמן טעינת האתר, ככל שהקובץ קטן יותר זמן הטעינה מהיר יותר.

על מנת שנוכל להשתמש בGrunt, כמו שציינתי מדובר בחבילה שנמצאת בתוך Nodejs ולכן נצטרך להרים שרת nodejs.
הקמתם סביבת עבודה של Nodejs (לדוגמה Express)? יופי! בואו נמשיך...

נתחיל בהתקנת Grunt על שרת הNodejs שלנו, נפתח CMD על התיקיה של השרת ונריץ את השורות הבאות (במק להוסיף sudo לפני npm):

npm install -g grunt-cli // install the package of grunt.
npm init -f // create/update your package.json with grunt properties

אחרי שהתקנו את grunt, נתקין את התוסיפים שבהם נשתמש על השרת שלנו.
נריץ את השורות הבאות בCMD :

npm install grunt-contrib-watch --save --dev
npm install grunt-contrib-htmlmin --save --dev

על מנת שנוכל להשתמש בgrunt ולהפעיל את שתי התוספים שאנחנו נעבוד איתם היום ניצור קובץ בתיקיה הראשית (איפה שנמצא package.json) בשם Gruntfile.js, זהו הקובץ שיחיל את כל ההגדרות של התוספים.

בתוך קובץ Gruntfile.js נכניס את הקוד הבא 


    module.exports = function(grunt) {

        grunt.initConfig({
            htmlmin: {
                dist: {
                    files: {
                        'public/index.min.html': 'public/index.html', // 'destination': 'source'
                    }
                }
            },
            watch: {
                files: ['public/index.html'],
                tasks: ['htmlmin']
            }
        });
        // Load the plugins that provides the "htmlmin" and "watch" tasks.
        grunt.loadNpmTasks('grunt-contrib-watch');
        grunt.loadNpmTasks('grunt-contrib-htmlmin');

        grunt.registerTask('default', ['watch']);

    };

initConfig מדובר בפונקציה שמקבלת אובייקט, שבמקרה הזה מכילה את ההגדרות של התוספים שאנחנו רוצים להריץ על השרת, נסביר קצת מה רשמנו בתוך האובייקט: ניתן לראות שישנם עוד 2 אובייקטים עיקריים, htmlmin ו watch.

htmlmin מקבל הגדרה של קובץ אליו אנחנו רוצים להפוך את הקובץ ל minify - להקטין את גודל הקובץ כמה שניתן לצורכי אופטימיזציה לטעינה מהירה של האתר - (index.min.html).

ו index.html זה קובץ המקור שאותו אנחנו מבקשים לכווץ בעזרת התוסף htmlmin.

באובייקט watch אנחנו בוחרים את הקובץ עליו אנחנו מעוניינים לעקוב ובמידה וחל שינוי, באופן אוטומטי Grunt ידע להפעיל את התוסף htmlmin (ניתן לראות שיש קריאה בtasks לתוסף).

בחלק האחרון של הקוד ניתן לראות שאנחנו מבצעים טעינה של התוספים בתוך המודול על מנת שנוכל להשתמש בתוספים:

grunt.loadNpmTasks('grunt-contrib-watch');
grunt.loadNpmTasks('grunt-contrib-htmlmin');

ברגע שנרשום את המשימה במודול כשנריץ בcmd: grunt אוטומטית grunt יפעיל את התוסף watch.
בלי שנרשום את התוסף, watch לא יפעל ולא יעקוב אחרי שינויים בקבצים.

grunt.registerTask('default', ['watch']);

הפעלת grunt דרך הcmd תגרום לwatch לפעול ולעקוב אחרי כל שינוי שיחול בקובץ index.html ויכווץ אותו לindex.min.html.

ישנם עוד אלפי תוספים שימושיים שיקלו עליכם מצרף לכאן קישור לתוספים הכי פופולארים בgrunt.
