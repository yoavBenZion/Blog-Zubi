JavaScript Hoisting

מי שמכירJS יודע שישנם מקרים שJS בנויה ומתנהגת שונה מאשר שאר שפות התכנות. אחד מהמקרים הללו הוא HOISTING. מנגנון בJavaScript  הפועל מאחורי הקלעים שמעביר את כל ההצהרות על המשתנים והפונקציות לראש התחום אליהם הם שייכים.

לכן על מנת למנוע בעיות בקוד רצוי תמיד להצהיר על משתנים בראש הדף ובפונקציות להצהיר ולאתחל משתנים לפני שימוש בהם.

Hoisting and variables:

נתחיל מהדוגמא הפשוטה ביותר:
הגדרת משתנה בראש הדף ושימוש באותו המשתנה על מנת להבין על מה מדובר הינה דוגמה שממחישה את ההשפעה של hoisting על משתנים:

var myvar = 'my value';
alert(myvar); // my value

כמובן שזה יציג 'my value’ בהודעת אזהרה קופצת על גבי המסך.. נמשיך לקוד הבא וניצור פונקציה פשוטה..

בשלב הבא נעטוף את הפונקציה alert בפונקציה אנונימית ולא אמורה להיות שום השפעה על התוצאה

var myvar = 'my value';
(function () {
alert(myvar); // my value
})();

ברור זה יעבוד כמו שצריך, בסך הכל יצרנו פונקציה פשוטה.

נוסיף עוד הצהרה על myvar ונאתחל אותה מתחת לalert. כבר נתחיל לראות איך Hoisting פועל מאחורי הקלעים בJavaScript.

var myvar = 'my value';
(function () {
alert(myvar); // undefined
var myvar = 'local value';

})();

ניתן לראות שHoisting העלה לראש הפונקציה את ההצהרה בלבד על המשתנה המקומי שיצרנו ואת האתחול של המשנה השאיר באותו מקום.

var myvar = 'my value';
(function () {
var myvar; // declare
alert(myvar); // undefined
myvar = 'local value'; // initialization

})();

Hoisting and functions:

בשונה משפות תכנות אחרות שיש להצהיר ולאתחל את הפונקציה לפני שימוש בJavaScript אם נציב את מימוש הפונקציה לפני שהגדרנו את הפונקציה עצמה, לא נקבל שגיאה או הערה undefined, המימוש יעבוד!

לגבי ההשפעה של hoisting על פונקציות? ההתנהגות בהקשר לפונקציות זהה לזו של hoisting על משתנים וגם מעלה מחוץ לתחום הפונקציה את המימוש של פונקציה:

לדוגמה:

(function isItHoisted () {
console.log('yes');
})();

אחרי Hoisting:

isItHoisted ();
(function isItHoisted () {
console.log('yes’); // will work!!
})();
